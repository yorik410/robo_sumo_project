# Проект робота для соревнований робо-сумо

Репозиторий содержит программы для управления роботом, участвующим в соревнованиях по робо-сумо. 
Здесь находятся основная программа, дополнительные стратегии, утилиты калибровки и тестовые программы.

## Структура проекта

Все дополнительные программы расположены в одноимённых папках.

```
ROOT
│
├── README.md
│
├── main                     # основная программа
├── calibration_for_stm32    # программа калибровки
│
├── strategies
│   ├── RUSH                 # стратегия движения напролом
│   └── without_sensors      # стратегия без дальномеров
│
└── tests
    ├── motors
    └── motors_by_one
```

### Основная стратегия
Это стандартная программа для робота в обычной конфигурации. Этапы ее стратегии:
1) Нахождение противника - вращение вокруг своей оси, пока противник не будет обнаружен
2) Атака - движение в сторону противника, а если его все еще видно, то скорость движения увеличивается
3) При попадании на линию - как-либо отъезжать от нее

### Калибровка
Программа калибрует только аналоговые датчики черной линии. Простейшая калибровка. Записывает данные о максимальных значениях на черном цвете. Во всех программах определение черной линии осуществляется по формуле
```
cur - max + 200 >= 0 
```
где:

`cur` — текущее значение датчика  
`max` — значение, полученное при калибровке

200 выступает в роли порогового значения, в рамках которого интерпретируются показания датчиков.

Данные для калибровки записываются в EEPROM в первые `2*n` значений, где `n` - кол-во датчиков для калибровки.

### Другие стратегии
#### RUSH
В этой стратегии робот движется просто прямо на максимальной скорости, но отъезжая от черной линии. Можно не использовать логику отъезда, если датчики не работают, в таком случае робот точно упадет, но может забрать с собой противника.

#### Без датчиков
В этой стратегии не используются дальномеры. Схожа по принципу работы с RUSH, но более сложная логика отъезда от черной линии. Стратегия может применяться как альтернативная основной или для ситуаций, когда дальномеры не работают.

### Тестовые программы
Простые программы, где проверяется только один элемент программы/конструкции.
#### motors
Простой запуск движения. Переключение движения через bluetooth.
#### motors by one
То же самое, что и `motors`, но запускается только один по упрощенной схеме. Автоматической смены движущегося мотора не предусмотрено.

## Конфигурация
Это стандартная конфигурация для этого проекта, можно использовать почти любую другую, но тогда может требоваться модификация.
### Механика
Робот трехколесный. Два задних колеса соосны и являются ведущими. Третье колесо - шарик. На ведущие колеса установлен понижающий редуктор с ременной передачей. 
Корпус не закрытый из-за больших размеров, но рекомендуется делать более закрытые варианты для некоторой противоударной защиты.
Защита спереди была представлена большим ковшом, в который встроен ультразвуковой датчик. Сверху ковша был установлен лазерный дальномер.
Датчики черной полосы стояли в передних углах робота.
### Электроника 
```
Микроконтроллер: STM32 Nucleo-32 (STM32F303K8T6)
Датчики черной линии: Pololu qtr-1A (аналоговые)
Лазерный дальномер: VL53L0X-v2
Ультразвуковой датчик: HC-SR04
Bluetooth-модуль: HC-05
Моторы: коллекторные мотор-редукторы 12В 37 мм (Китай)
Драйвера моторов: VNH3SP30
Питание: Литий-ионный аккумулятор 12В
```

Все подключалось на печатную плату с распределением питания и прочим.

Лазерный дальномер на шине I2C. Ультразвуковой датчик работает от 5В, но все остальное от 3.3В. Bluetooth-модуль использовался в основном для калибровки.
В основной программе необходимо наличие корректного расстояния на выходе хотя бы одного из двух дальномеров. 

HC-SR04 может плохо определять наклонные поверхности или вовсе их не видеть.

## Программа
Программа написана для использования с `stm32duino`, точно работает для версии `2.7.1 stm32duino`, другие версии могут не работать.

В основной программе используются библиотеки `Wire`, `EEPROM`, `VL53L0X`. Первые две стандартные, третья устанавливается дополнительно от издателя `Pololu`, версии `1.3.1`, другие версии не тестировались.

## Сборка и загрузка

1. Установить Arduino IDE (возможны альтернативы)
2. Установить пакет STM32duino
3. Открыть проект из папки `main`
4. Выбрать плату:
   `Nucleo-32 F303K8`
5. Загрузить прошивку через USB (Mass Storage)
